## Change 1 — Expand CI trigger for feature branches

Update `.github/workflows/ci.yml` `on:` block to:

```yaml
on:
  push:
    branches:
      - main
      - 'feat/**'
      - 'fix/**'
      - 'refactor/**'
      - 'chore/**'
  pull_request:
    branches: [main]
```

Rationale: matches existing commit-message convention
(feat/fix/refactor/chore prefixes). CI runs on branch pushes
before merge, without requiring PRs. Direct pushes to main and
PRs to main also trigger CI. No-PR workflow still holds.
